# КЧД и метод Райта: пошаговый разбор

---

## Часть 1. Красно-чёрное дерево

### Что такое КЧД

Красно-чёрное дерево (КЧД) — сбалансированное бинарное дерево поиска, где каждый узел покрашен в красный или чёрный цвет. Балансировка достигается не точным выравниванием, а поддержанием четырёх инвариантов:

1. Корень — **чёрный**.
2. Красный узел не имеет красного родителя (нет двух красных подряд).
3. Любой путь от корня до NIL-листа содержит одинаковое число чёрных узлов («чёрная высота»).
4. NIL-листья считаются чёрными.

Из этих правил следует: самый длинный путь ≤ 2 × кратчайшего → высота O(log n) → поиск, вставка, удаление за O(log n).

### Три ситуации при вставке

Новый узел всегда вставляется **красным** (чёрная высота не меняется). Затем запускается `fixup`, который разрешает конфликт «красный под красным»:

| Ситуация | Дядя | Действие |
|---|---|---|
| 1 | Красный | Перекраска дяди и родителя → чёрные, деда → красный; поднимаемся вверх |
| 2 | Чёрный, z — «внутренний» ребёнок | Поворот вокруг родителя → сводим к ситуации 3 |
| 3 | Чёрный, z — «внешний» ребёнок | Поворот вокруг деда + перекраска; конфликт устранён |

---

### Пример: вставка 1, 2, 3, 4, 5, 6

#### Шаг 1 — вставляем 1

1 — первый узел, становится корнем. Правило 1: корень всегда чёрный.

```
1(Ч)
```

---

#### Шаг 2 — вставляем 2

2 > 1 → правый ребёнок. Новый узел красный. Родитель (1) **чёрный** → нарушений нет.

```
  1(Ч)
    └─П: 2(К)
```

---

#### Шаг 3 — вставляем 3

3 > 2 → правый ребёнок 2. Родитель (2) **красный** — конфликт!
Дядя — NIL (чёрный). Узел 3 — правый ребёнок правого родителя → **ситуация 3**.

Действия:
- 2.цвет = Чёрный
- 1.цвет = Красный
- `rotL(1)` — левый поворот вокруг 1: узел 2 поднимается наверх

**До поворота:**
```
1(Ч)
  └─П: 2(К)
        └─П: 3(К)  ← новый
```

**После `rotL(1)` + перекраска:**
```
    2(Ч)
   /    \
1(К)   3(К)
```

Корень 2 красим в чёрный → правило 1 выполнено.

---

#### Шаг 4 — вставляем 4

4 > 3 → правый ребёнок 3. Родитель (3) **красный** — конфликт!
Дядя — узел 1, цвет **красный** → **ситуация 1**.

Действия:
- 3.цвет = Чёрный, 1.цвет = Чёрный (оба дети → чёрные)
- 2.цвет = Красный (дед → красный)
- 2 — корень → снова чёрный (правило 1)

```
      2(Ч)
     /    \
  1(Ч)   3(Ч)
            └─П: 4(К)
```

---

#### Шаг 5 — вставляем 5

5 > 4 → правый ребёнок 4. Родитель (4) **красный** — конфликт!
Дядя — NIL (чёрный). Узел 5 — правый ребёнок правого родителя → **ситуация 3**.

Действия:
- 4.цвет = Чёрный
- 3.цвет = Красный
- `rotL(3)` — левый поворот вокруг 3: узел 4 поднимается

**До поворота:**
```
2(Ч)
 └─П: 3(Ч)
        └─П: 4(К)
               └─П: 5(К)  ← новый
```

**После `rotL(3)` + перекраска:**
```
      2(Ч)
     /    \
  1(Ч)   4(Ч)
         /   \
       3(К) 5(К)
```

---

#### Шаг 6 — вставляем 6

6 > 5 → правый ребёнок 5. Родитель (5) **красный** — конфликт!
Дядя — узел 3, цвет **красный** → **ситуация 1**.

Действия:
- 5.цвет = Чёрный, 3.цвет = Чёрный
- 4.цвет = Красный (дед → красный)
- 4.родитель = 2 (чёрный) → fixup завершается, нарушений нет
- Корень (2) остаётся чёрным

**Итоговое дерево:**
```
          2(Ч)
         /    \
      1(Ч)   4(К)
             /   \
          3(Ч)  5(Ч)
                  └─П: 6(К)
```

**Inorder-обход (лево → узел → право): 1, 2, 3, 4, 5, 6** ✓

### Сводная таблица вставок

| Шаг | Узел | Ситуация | Действие |
|-----|------|----------|----------|
| 1 | 1 | Корень | Красим в чёрный |
| 2 | 2 | Родитель чёрный | Ничего |
| 3 | 3 | Дядя чёрный, внешний | `rotL(1)` + перекраска |
| 4 | 4 | Дядя красный | Перекраска 1, 3, 2 |
| 5 | 5 | Дядя чёрный, внешний | `rotL(3)` + перекраска |
| 6 | 6 | Дядя красный | Перекраска 3, 5, 4 |

## Часть 2. Метод Райта (lr3, поиск в отсортированном двусвязном списке)

### Идея

Метод Райта — поиск ключа в **отсортированном двусвязном списке (деке)**, который стартует с середины списка, а не с головы или хвоста. Это даёт ~n/4 сравнений в среднем вместо ~n/2 при линейном поиске с головы.

Структура: каждый узел хранит `value`, указатели `next` и `prev`. Список отсортирован по возрастанию.

### Три шага алгоритма

**Шаг 1 — переход на середину.**
Проходим `size/2` шагов по `next` от `front`. Получаем указатель `cur` на средний узел. Это O(n/2), но выполняется один раз.

**Шаг 2 — сравнение с серединой.**
```
если cur->value == key → нашли, возвращаем cur
```

**Шаг 3 — движение в одну сторону.**
```
если key < cur->value:
    пока cur->prev != NULL и cur->value > key:
        cur = cur->prev
    если cur->value == key → нашли

если key > cur->value:
    пока cur->next != NULL и cur->value < key:
        cur = cur->next
    если cur->value == key → нашли
```

Итоговая финальная проверка: `cur->value == key`.

### Почему ~n/4 сравнений

Прыжок на середину делит список пополам. После этого ключ ищется только в одной половине — влево или вправо. В среднем по этой половине пройдём ещё n/4 шагов. Итого: 1 (средний) + n/4 = **~n/4 сравнений**.

### Пример

**Список (n = 8):** `2 ↔ 5 ↔ 9 ↔ 14 ↔ 17 ↔ 21 ↔ 28 ↔ 35`

Ищем ключ **21**.

**Шаг 1 — переход на середину:**
`size/2 = 4` шага от `front (2)`: 2 → 5 → 9 → 14 → **17**
`cur = 17`

**Шаг 2 — сравнение с серединой:**
`17 == 21`? Нет.

**Шаг 3 — движение вправо** (21 > 17):
```
cur->next = 21: 21 < 21? Нет → стоп
cur = 21
```
`21 == 21`? **Нашли!**

Итого сравнений: 1 (середина) + 1 (шаг вправо) + 1 (финал) = **3** вместо 6 при линейном поиске с головы.

---

**Теперь ищем ключ 5** в том же списке.

**Шаг 1:** `cur = 17` (середина)

**Шаг 2:** `17 == 5`? Нет.

**Шаг 3 — движение влево** (5 < 17):
```
cur->prev = 14: 14 > 5 → cur = 14
cur->prev = 9:  9 > 5  → cur = 9
cur->prev = 5:  5 > 5? Нет → стоп
cur = 5
```
`5 == 5`? **Нашли!**

Итого сравнений: 1 + 3 + 1 = **5** (вместо 2 с головы — тут линейный был бы быстрее, но в среднем по всем ключам метод выигрывает).

### Сравнение с линейным поиском

| Критерий | Линейный (с головы) | Метод Райта |
|----------|---------------------|-------------|
| Лучший случай | 1 | 1 (ключ = середина) |
| Средний случай | ~n/2 | ~n/4 |
| Худший случай | n | ~n/2 |
| Требования | Нет | Список отсортирован |
| Применение в lr | — | lr3 (деки, `deque.c`) |

---

## Итог

КЧД при вставке последовательности 1..6 выполнил **два левых поворота** (шаги 3 и 5) и **две перекраски** (шаги 4 и 6). Дерево всегда оставалось валидным: чёрная высота = 2 на протяжении всей работы.

Метод Райта сокращает среднее число сравнений вдвое относительно линейного поиска за счёт прыжка на середину отсортированного списка и последующего движения только в одном направлении.
